HTML5中的History API

发表于 2018-12-12 | 分类于 JavaScript

History API是用于操纵浏览器中的会话历史记录的API

属性和方法

  • history.length

    返回一个整数，表示会话历史的数目，包括当前页面。

  • history.back()

    返回上一页，相当于history.go(-1)

  • history.forward()

    前往下一页，相当于history.go(1)

  • history.go(num)

    传入一个整数，前往从历史中相对于当前位置的页面，整数超出历史记录界限时没有效果也不会报错

  • history.pushState(state, title, url)

    • 往历史记录堆栈的顶部添加一个状态，传入三个参数，第一个为状态对象，第二个为标题（目前浏览器会忽略该参数），第三个为地址。state可为一个对象或者null

    • 当state参数序列化后超过640k时会抛出异常。

    • url需要和当前URL在同一个源下（即不允许跨域），否则也会报错
    • 该方法调用后就可以通过history.state获取state对象

  • history.replaceState(state,title,url)

    替换当前页面的历史记录

以上两种方式pushState以及replaceState都只更改记录的值，而不真正去加载传入的URL

  • onpopstate事件

    每当同一个文档的浏览历史（即history对象）出现变化时，就会触发popstate事件。

    仅仅调用pushState()方法或replaceState()方法 ，并不会触发该事件，只有用户点击浏览器倒退按钮和前进按钮，或者使用 JavaScript 调用History.back()、History.forward()、History.go()方法时才会触发。另外，该事件只针对同一个文档，如果浏览历史的切换，导致加载不同的文档，该事件也不会触发。onpopstate事件的两种回调方法：
